# Descripción de la práctica

Esta práctica introduce el uso básico de **Git** y **GitHub**, trabajando con archivos **Markdown (.md)**. El objetivo principal es aprender a gestionar repositorios, utilizar ramas y practicar la creación de *Pull Requests*.

## Objetivos

- Aprender los comandos principales de **Git**.  
- Practicar la creación y fusión de **ramas**.  
- Visualizar el historial de **commits** y ramas mediante un gráfico.  
- Aprender a crear y gestionar una **Pull Request**.

## Pasos principales

1. Crear y clonar un repositorio de GitHub en tu ordenador.  
2. Añadir un archivo `README.md` con tu información y la descripción de la práctica.  
3. Crear dos ramas: `feature/ShoppingCard` y `feature/ShowProduct`.  
4. Añadir contenido específico a cada rama.  
5. Crear un conflicto de fusión y resolverlo.  
6. Crear, revisar y fusionar las **Pull Requests**.  
7. Publicar la URL del repositorio final en GitHub.

## Resultado final

El resultado será un repositorio con el historial completo de commits, ramas y fusiones, mostrando un buen uso de Git y GitHub para la colaboración y el control de versiones.
